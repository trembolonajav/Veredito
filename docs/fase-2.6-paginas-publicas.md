# Fase 2.6 — Páginas Públicas Principais

Status: **pendente** (após Fase 2.5)

---

## Objetivo

Construir todas as páginas públicas do portal com dados mockados.

---

## Ordem de implementação

### 2.6.1 — Home

- Hero com manchete principal (imagem + título + editoria + autor)
- Grid de destaques secundários (3-4 cards)
- Seção por editoria (2-3 editorias com 3 cards cada)
- Bloco newsletter
- "Últimas notícias" com listagem cronológica

### 2.6.2 — Página de conteúdo individual

Base compartilhada com variações por tipo:

**Layout padrão:**
- Breadcrumb
- Eyebrow (editoria + tipo)
- Título (Instrument Serif, grande)
- Subtítulo
- Meta (autor + data + tempo de leitura)
- Imagem de capa com legenda
- Corpo do texto
- Tags
- Box do autor
- "Leia também"

**Variações:**
- Decisão: banner com tribunal + processo + relator + tese fixada
- Opinião: disclaimer + avatar do autor proeminente
- Artigo: referências bibliográficas no final

### 2.6.3–2.6.6 — Listagens por tipo

Cada listagem segue o mesmo padrão:
- Header com título da seção
- Filtros por editoria
- Grid de cards (3 colunas desktop, 1 mobile)
- Paginação
- Sidebar com "Mais lidos" e "Newsletter"

### 2.6.7 — Página de editoria

- Header com nome + descrição da editoria
- Últimos conteúdos dessa editoria
- Autores que escrevem nessa editoria

### 2.6.8 — Página do autor

- Avatar + nome + bio
- Contagem de artigos
- Listagem de conteúdos do autor

### 2.6.9 — Busca

- Campo de busca grande
- Resultados com highlight do termo
- Filtros: tipo, editoria, data

### 2.6.10–2.6.13 — Páginas institucionais

- **Sobre**: história, missão, equipe
- **Contato**: formulário simples
- **Newsletter**: landing page de inscrição
- **Privacidade / Termos**: texto legal

---

## Critérios de aceite

- [ ] Portal parece um produto real
- [ ] Navegação completa entre páginas
- [ ] Conteúdo circula bem (links cruzados)
- [ ] Responsivo em todos os breakpoints
- [ ] Tipografia editorial consistente
